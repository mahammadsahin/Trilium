File System & File Format

All Linux follow “Filesystem Hierarchy Standard (FHS)”

A file system is basically a set of rules used to decide how data is stored and fetched in a storage device, be it a hard drive, flash drive, or something else.

 

NTFS V/S FAT V/S EXT
  • FAT-32
    • File allocation table
    • Support up to 2TB
    • 1 file 4gb max file size
  • NTFS
    • New Technology File System
    • Support 16 tb max file size
  • EXT4
    • Extended file system 4
    • File based encryption and jurnalling (error control)
    • Ext4 can access NTFS file system

[*] In one machine 1 hdd/ssd create max 4 partition - 3 primary & 1 extended then 1 extended to 15 logical partition we create

 

File system
  • / - Root / Top Level Directory
    • /bin - user executable binaries files/Application Binary - System Software, Shell command (apt, ls, cd, cat)
    • /boot – boot loader & Kernel Files
    • /dev – device files for every hardware devices attached to the system
    • /dev/sda - Where install linux like a harddisk then separate different partation like /dev/sda1(boot), /dev/sda2, /dev/sda3
    • /etc – configuration files 
    • /home – users files 
    • /lib - Shared library for boot required files & /bin softwares
    • /media – external media temp mount point 
    • /mnt – Temporary mount point for file system 
    • /opt – optional files of application 
    • /proc
    • /root – root user 
    • /run
    • /sbin – assiontial binaryes [system administrator- Fdisk, mkfs, sysctl]
    • /srv - Data for services provided by this system. This directory gives users the location of data files for a particular service, such as FTP, WWW)
    • /sys
    • /tmp – Temp files 
    • /usr/ - Shareable, read-only, externally install software, executable binary files and libraries, man files, and other types of documents - (i) second level program, (ii) apt get install, (iii) git clone
      • /bin 
      • /include 
      • /lib 
      • /sbin 
    • /var/ - variable – which includes spool directories and files, logging data, transient and temporary files, server data, web server files/login data , logs
